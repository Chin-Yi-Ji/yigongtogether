# YigongHub 專案狀態：未完成與後續

最後整理：依 README 與程式現況整理。

---

## ✅ 目前已完成

- Google OAuth 登入、建立 Profile（onboarding）
- 首頁（介紹 + 登入/建立帳號 + 聯絡開發者）
- 探索頁（hashtag 搜尋、熱門搜尋推薦、最新更新的 Profile、**後端全文搜尋**、**與你相似的人**）
- 個人頁 `/u/[handle]`（目前工作經歷 / 過往經歷、hashtag、對未來的期待、**依隱私設定顯示 location/經歷**）
- Prisma Schema（User、Profile、Experience、Tag、ProfileTag）
- Seed 假資料（含工作經歷）、Zeabur 部署用 Prisma 設定
- 部署到 Zeabur、同專案 PostgreSQL 連線、自訂網域說明
- UI（Tailwind v4、簡潔版型、RWD）、404 頁、探索頁 DB 錯誤時仍可顯示
- **Profile 編輯**：`/settings` 頁、PATCH API、基本資料／標籤（興趣／專長／主題／hashtag）／隱私設定
- **Experience 動態表單**：設定頁可新增／刪除經歷，API：POST、PATCH、DELETE
- **隱私設定 UI**：設定頁可設 location／經歷為「公開／僅登入／不顯示」，個人頁依權限顯示
- **Dark mode 切換**：Header 主題切換按鈕，localStorage 持久化
- **中文 hashtag**：`normalizeTag` 支援中文（CJK），seed 與 API 一致
- **Tag 類型**：interest / expertise / topic / hashtag，設定頁四欄、個人頁顯示類型
- **API 輸入驗證**：Profile create/update、Experience create/update 使用 Zod 驗證

---

## Handle 無法變更（產品決定）

個人連結名稱（handle）**第一次在 onboarding 填寫後即無法修改**，以維持個人頁網址穩定。Onboarding 與設定頁均已註明此說明。

## ⏳ 可再補強（非必須）

| 項目 | 說明 |
|------|------|
| **全文搜尋進階** | 已用 Prisma `contains` 並支援 handle、tagline；若資料量變大可考慮 PostgreSQL pg_trgm 或 Meilisearch。 |
| **錯誤／403 頁** | 未登入訪問受保護頁可再補專用 403 頁；API 錯誤可統一前端 toast。 |

### 已完成的維運、安全與進階功能

- **Rate limiting**：已實作（登入、建立/更新 Profile、上傳大頭照、刪除帳號申請），見 `src/lib/rate-limit.ts`、`src/middleware.ts`。
- **頭像上傳**：已支援本機與 Cloudflare R2（見 docs/Cloudflare-R2-大頭照設定.md）。
- **搜尋體驗**：探索頁搜尋已包含 handle、displayName、oneLiner、tagline、標籤名稱。
- **SEO / metadata**：個人頁（動態 title/description）、探索頁、設定頁、關於頁均已設定 metadata。
- **表單驗證**：設定頁與 onboarding 已加入即時欄位驗證與錯誤訊息（顯示名稱、一句話、自我介紹、個人連結格式等）。
- **數據與統計**：管理後台新增「數據與統計」區塊（總帳號數、已建個人檔案、待確認刪除）。
- **標籤管理**：管理後台新增「標籤管理」、可隱藏標籤（隱藏後不顯示於探索熱門搜尋）；Tag 模型已加 `hidden` 欄位，部署後需執行 `npm run db:push`。
- 部署與文件以 Zeabur 為主；若之後要加其他平台可再補。

其餘可再補強項目可依實際使用再排。更完整清單見 [改善與新增功能分析.md](改善與新增功能分析.md)。
